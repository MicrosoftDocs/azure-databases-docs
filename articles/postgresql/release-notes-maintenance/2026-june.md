# Azure Database for PostgreSQL flexible server: June 2026 release

We're excited to announce the June 2026 version of Azure Database for PostgreSQL. Beginning June 5, 2026, the service automatically onboards all new servers to this latest version. The service upgrades existing servers during their next scheduled maintenance.

This new version introduces a range of new features and enhancements, resolves known issues, and includes essential security patches to ensure optimal performance and security.

## Engine version changes

- Latest minor versions updated for Azure Database for PostgreSQL: [18.4, 17.10, 16.14, 15.18, and 14.23](https://www.postgresql.org/about/news/postgresql-184-1710-1614-1518-and-1423-released-3297/)

## Features

- Support for pre-Upgrade validation checks prior to Major Version Upgrade - **Public Preview**
- Major Version Upgrade support for PG18 released worldwide
- Added Major Version Upgrade support to PostgreSQL 17 and PostgreSQL 18 for customers using ephemeral tablespaces.
- Onboarded [temporal_tables](https://github.com/arkhipov/temporal_tables) extension
- Added support for DROP CAST


## Improvements 

- **Major Version Upgrade** - Improved upgrade reliability for DBs with a high number of large objects (LOs), reducing the risk of upgrade failures due to memory pressure.
- Improved failover and scale operation reliability by reducing vCore gateway switchover time from up to 5 minutes to less than 1 minute, helping minimize customer downtime.

## Known issues fix

- No known issue fixes
