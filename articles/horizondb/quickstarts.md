# Create an Azure HorizonDB Instance

Azure HorizonDB is Azure’s next generation, cloud native PostgreSQL service, featuring disaggregated storage, ultra low latency WAL writes, elastic read scale-out replicas, and built in AI & vector capabilities. It's build for business, offering comprehensive security, high availability, workload scalability, and disaster recovery for enterprise applications, while also being engineered for developers with industry leading AI features, advanced vector indexing, first class VS Code integration, and seamless mirroring to Microsoft Fabric for near real time analytics
This quick start walks you through creating a HorizonDB instance and connecting to it using Azure portal, Azure CLI (TBD), or Azure Resource Manager (TBD).
Prerequisites
Before you begin, make sure you have:
•	An Azure subscription. If you don't have one, create a free Azure account.
•	Appropriate permissions to create resources in your subscription.
Note – Azure HorizonDB is available in limited regions during the public preview. See complete list of all the regions where Azure HorizonDB is currently available. 
A HorizonDB instance features a disaggregated compute and storage architecture with built-in high availability through zone redundancy, automatic read scaling with up to 15+ replicas, and support for AI ready extensions such as pgvector, azure_ai, and pg_diskann (preview).
